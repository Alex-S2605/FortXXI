Подзадачи 1, 2 реализованы в файле
proc_to_udp.c - Консольный UDP-client

Протокол передачи:
"BEGIN" - 5 байт - признак начала;
uint32_t message_counter - 4 байна - счётчик сообщений;
uint16_t cpu_count - 2 байта - число ядер;
1/0 - 1 байт - Признак валидности информации общей загрузки (1 валидно);
float total_usage - 4 байта - Суммарная загрузка процессора;
Для кажного ядра
1/0 - 1 байт - Признак валидности информации загрузки ядра (1 валидно);
float cpu_usage - - 4 байта - Звгрузка ядра;
"END" - 3 байта - признак окончания.

Подзадача 3 реализована в проекте ProcStatPlot.pro.

Допущения:
Число ядер в процессе работы программ не меняется.
Разрыва датаграмм при считывании не происходит (Для обработки ситуации в протокол заложено "BEGIN" - "END").
UDP-client работает без перерывов (Для обработки пропусков сообщений и перерывов в работе UDP-client в протокол заложен счётчик сообщений).
Обработки сбоев поступления данных с UDP не проводится.

Разработка осуцествлялась в ОС Windous и Ubuntu.
